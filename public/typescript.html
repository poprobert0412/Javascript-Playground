🔷 Introducere în TypeScript

TypeScript = JavaScript + tipuri statice. Același cod JS pe care îl cunoști, dar cu superputeri: detectează erorile înainte de a rula codul!

🔷 TypeScript 5.x — Superset of JavaScript
1
Ce este TypeScript?
Bazele TS

TypeScript este un superset al JavaScript-ului, creat de Microsoft. Asta înseamnă că orice cod JavaScript valid este și cod TypeScript valid. TypeScript adaugă tipuri statice — adică spui explicit ce tip de date conține fiecare variabilă.

🎓 De ce contează? JavaScript nu te oprește dacă faci let x = 5; x = "text"; — dar asta poate cauza bug-uri subtile. TypeScript te avertizează înainte de a rula codul!
JavaScript let nume = "Robert"; nome = 42; // ✅ JS permite, dar e bug!
TypeScript let nume: string = "Robert"; nume = 42; // ❌ Error: Type 'number' // is not assignable to type 'string'
2
Tipuri de bază (Primitive Types)
string · number · boolean · null · undefined

TypeScript are aceleași tipuri primitive ca JavaScript, dar le poți declara explicit:

// Tipuri primitive let nume: string = "Robert"; let varsta: number = 25; let student: boolean = true; let nimic: null = null; let nedefinit: undefined = undefined; // Array-uri tipizate let numere: number[] = [1, 2, 3]; let nume: string[] = ["Ana", "Ion"]; // Tuple — array cu tipuri fixe let pereche: [string, number] = ["Robert", 25];
💡 Type Inference: TypeScript poate ghici tipul automat! let x = 5 → TS știe că x e number fără să spui explicit. Dar e bine să le declari când codul e complex.
3
Funcții cu tipuri
function types · return types · optional params

În TypeScript, parametrii și return-ul funcțiilor au tipuri. Asta previne erori ca: "am trimis un string unde trebuia un număr".

JavaScript function aduna(a, b) { return a + b; } aduna("5", 3); // "53" 😱 string!
TypeScript function aduna(a: number, b: number): number { return a + b; } aduna("5", 3); // ❌ Error!
// Parametri opționali (?) function salut(nume: string, titlu?: string): string { if (titlu) return `Salut, ${titlu} ${nume}!`; return `Salut, ${nume}!`; } salut("Robert"); // ✅ "Salut, Robert!" salut("Robert", "Dl."); // ✅ "Salut, Dl. Robert!" // Valori default function putere(baza: number, exp: number = 2): number { return baza ** exp; }
4
Interfaces — Structuri de date
interface · shape of objects

O interfață definește "forma" unui obiect — ce proprietăți trebuie să aibă și de ce tip sunt. E ca un contract: "orice obiect de acest tip TREBUIE să aibă aceste câmpuri".

// Definim interfața interface User { nume: string; varsta: number; email: string; premium?: boolean; // opțional } // Folosim interfața const user: User = { nume: "Robert", varsta: 25, email: "robert@mail.com" }; // ✅ OK! const bad: User = { nume: "Ion" }; // ❌ Error: lipsește 'varsta' și 'email' // Funcție care acceptă un User function salutaUser(u: User): string { return `Salut, ${u.nume}! Ai ${u.varsta} ani.`; }
5
Type Aliases & Union Types
type · union (|) · literal types

Type alias = un nume personalizat pentru un tip. Union type (|) = o valoare poate fi de mai multe tipuri.

// Type alias type ID = string | number; let userId: ID = "abc-123"; // ✅ string userId = 42; // ✅ number userId = true; // ❌ Error! // Literal types — exact aceste valori! type Direction = "up" | "down" | "left" | "right"; function move(dir: Direction) { console.log(`Moving ${dir}`); } move("up"); // ✅ move("diag"); // ❌ Error: not in union // Union cu interfețe type Result = { success: true; data: string } | { success: false; error: string };
🤔 Interface vs Type? Ambele definesc forme de obiecte. Diferențe: interface poate fi extinsă cu extends, type poate face union (|) și intersection (&). Regulă: folosește interface pentru obiecte, type pentru union-uri și alias-uri.
6
Enums — Seturi de constante
enum · const enum

Enum (enumerare) definește un set de constante cu nume. Perfect pentru stări, roluri, direcții, etc.

// Numeric enum (default: 0, 1, 2...) enum Status { Pending, // 0 Active, // 1 Completed, // 2 Cancelled // 3 } let taskStatus: Status = Status.Active; console.log(taskStatus); // 1 // String enum (mai descriptiv) enum Role { Admin = "ADMIN", User = "USER", Guest = "GUEST" } function checkAccess(role: Role) { if (role === Role.Admin) { console.log("Full access!"); } } checkAccess(Role.Admin); // ✅ "Full access!"
7
Generics — Tipuri flexibile
generics · <T> · reusable types

Generics permit crearea de funcții și clase care funcționează cu orice tip, păstrând type safety. Gândește-te la ele ca la variabile pentru tipuri: <T>.

Fără Generics ❌ // Pierde tipul! Returnează 'any' function first(arr: any[]): any { return arr[0]; } let x = first([1,2,3]); // x e 'any' — TS nu știe tipul 😞
Cu Generics ✅ // Păstrează tipul! function first<T>(arr: T[]): T { return arr[0]; } let x = first([1,2,3]); // x e 'number' — TS știe! 🎯
// Generic mai complex: o pereche cheie-valoare interface Pair<K, V> { key: K; value: V; } const p1: Pair<string, number> = { key: "age", value: 25 }; const p2: Pair<number, boolean> = { key: 1, value: true };
8
Type Guards — Verificare la runtime
typeof · instanceof · in · narrowing

Type guards sunt verificări care ajută TypeScript să "îngusteze" (narrow) tipul unei valori. După o verificare, TS știe exact ce tip e.

// typeof guard function formatValue(val: string | number): string { if (typeof val === "string") { return val.toUpperCase(); // TS știe: val e string } return val.toFixed(2); // TS știe: val e number } // 'in' guard (verifică proprietăți) interface Dog { bark(): void; } interface Cat { meow(): void; } function makeSound(pet: Dog | Cat) { if ("bark" in pet) { pet.bark(); // TS știe: e Dog } else { pet.meow(); // TS știe: e Cat } }
9
Clase TypeScript
class · public · private · readonly · implements

TypeScript adaugă modificatori de acces la clasele JavaScript: public, private, protected, readonly.

interface Printable { print(): void; } class Student implements Printable { // Shorthand: parametrii din constructor devin proprietăți constructor( public name: string, private grade: number, readonly id: string ) {} print() { console.log(`${this.name} — nota ${this.grade}`); } getGrade(): number { return this.grade; // ✅ acces din interior } } const s = new Student("Robert", 10, "STU-001"); s.name; // ✅ public s.grade; // ❌ Error: 'grade' is private s.id = "X"; // ❌ Error: 'id' is readonly
10
Utility Types — Superputeri TS
Partial · Required · Pick · Omit · Record · Readonly

TypeScript vine cu tipuri utilitare built-in care transformă alte tipuri. Sunt extrem de utile în cod real:

interface User { name: string; email: string; age: number; } // Partial — toate câmpurile devin opționale function updateUser(user: User, updates: Partial<User>) { return { ...user, ...updates }; } updateUser(user, { age: 26 }); // ✅ doar 'age' // Pick — selectează doar câmpurile dorite type UserPreview = Pick<User, "name" | "email">; // = { name: string; email: string; } // Omit — exclude câmpuri type UserPublic = Omit<User, "email">; // = { name: string; age: number; } // Record — obiect cu chei și valori tipizate type Scores = Record<string, number>; const scores: Scores = { math: 95, js: 100 }; // Readonly — nu poți modifica const frozen: Readonly<User> = { name: "X", email: "x@x", age: 20 }; frozen.name = "Y"; // ❌ Error: readonly!
11
Async & Promises cu tipuri
Promise<T> · async/await · fetch typing

TypeScript tipizează și operațiile asincrone — funcțiile async returnează Promise<T>.

// Funcție async tipizată async function fetchUser(id: number): Promise<User> { const res = await fetch(`/api/users/${id}`); const data: User = await res.json(); return data; } // Folosire const user = await fetchUser(1); console.log(user.name); // TS știe: e string console.log(user.xyz); // ❌ Error: 'xyz' nu există pe User // Promise.all tipizat const [user1, user2] = await Promise.all([ fetchUser(1), fetchUser(2) ]); // Ambii sunt User!
12
Cum rulezi TypeScript?
tsc · tsconfig · compilation

Browser-ul NU înțelege TypeScript direct. Trebuie compilat în JavaScript cu tsc (TypeScript Compiler).

# Instalare globală npm install -g typescript # Verifică versiunea tsc --version # Compilează un fișier tsc hello.ts # → generează hello.js # Watch mode (compilează automat la salvare) tsc --watch hello.ts # Inițializează tsconfig.json tsc --init
💡 Alternativ: Poți folosi ts-node pentru a rula direct fișiere .ts fără compilare manuală: npx ts-node hello.ts
⚠️ Important: TypeScript există DOAR la compilare. La runtime, totul devine JavaScript normal. Tipurile sunt "șterse" — nu există overhead de performanță!
