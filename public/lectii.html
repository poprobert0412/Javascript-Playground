📦 Lecția 1 — Variabile let, const, var

📦 Ce sunt variabilele?

Variabilele sunt "cutii" în care stochezi informații. Ca un sertar cu etichetă — pui ceva înăuntru, iar eticheta e numele variabilei.

javascript // CONST — constantă, NU se poate reasigna const nume = "Robert"; const PI = 3.14159; // LET — se poate reasigna (schimba) let varsta = 25; varsta = 26; // ✅ OK varsta++; // ✅ OK — acum e 27 // VAR — versiunea VECHE, nu o mai folosi! var vechi = "nu folosi var";
🎓 Sfat de Prof
Regulă simplă: folosește const pentru tot. Numai când trebuie să schimbi valoarea, folosește let. Nu folosi niciodată var — are probleme cu "scope-ul" (vei învăța mai târziu).

Deschide F12 → Console și scrie acolo variabilele! Experimentează live!

🏷️ Lecția 2 — Tipuri de Date string, number, boolean, null, undefined

🏷️ Tipurile primitive din JavaScript

JavaScript are 7 tipuri de date. Cele mai importante 5:

javascript // STRING — text (între ghilimele sau backticks) const text = "Salut, lume!"; const altText = `Bună, ${text}`; // template literal // NUMBER — numere (întregi sau zecimale) const intreg = 42; const zecimal = 3.14; // BOOLEAN — adevărat sau fals const eStudnet = true; const eNoapte = false; // NULL — "nimic" (setat INTENȚIONAT) const nimic = null; // UNDEFINED — "nu e definit" (implicit) let ceva; // ceva = undefined // typeof — verifică tipul typeof "salut" // → "string" typeof 42 // → "number" typeof true // → "boolean" typeof null // → "object" (BUG din 1995!)
💡 De ce contează tipurile?
"5" + 2 = "52" (lipește text!) vs 5 + 2 = 7 (adună numere). JavaScript face conversii automate care pot fi surprinzătoare. De aceea TypeScript a fost creat — adaugă verificarea tipurilor!

🔀 Lecția 3 — Condiții if, else, switch, ternary

🔀 if / else if / else — Decizii în cod

Condițiile permit codului tău să ia decizii. "Dacă X, fă asta. Altfel, fă altceva."

javascript // IF / ELSE IF / ELSE const nota = 8; if (nota >= 9) { console.log("Excelent! 🏆"); } else if (nota >= 5) { console.log("Promovat! ✅"); } else { console.log("Nepromovat ❌"); } // TERNARY — if/else pe O LINIE const mesaj = nota >= 5 ? "Promovat" : "Nepromovat"; // SWITCH — pentru mai multe cazuri const zi = "Luni"; switch (zi) { case "Luni": console.log("Start de săptămână"); break; case "Vineri": console.log("Weekend aproape!"); break; default: console.log("Zi normală"); }
🎓 Când folosești ce?
if/else = pentru 2-3 condiții simple
ternary ? : = pentru condiții pe o singură linie (ex: atribuire)
switch = când ai MULTE cazuri posibile (ex: ziua săptămânii, meniu)

⚠️ Nu uita break în switch! Fără el, codul "cade" în următorul case!

🔄 Lecția 4 — Bucle for, while, for...of, for...in

🔄 Buclele — repetă codul automat

Buclele execută o bucată de cod de mai multe ori. Fără ele, ar trebui să copiezi codul manual.

javascript // FOR — cel mai comun loop // for (INIȚIALIZARE; CONDIȚIE; PAS) for (let i = 0; i < 5; i++) { console.log(i); // 0, 1, 2, 3, 4 } // WHILE — repetă CÂT TIMP condiția e true let x = 10; while (x > 0) { x -= 2; // x = x - 2 } // x este acum 0 // FOR...OF — parcurge un array (simplu!) const fructe = ["mar", "banana", "kiwi"]; for (const fruct of fructe) { console.log(fruct); // "mar", "banana", "kiwi" } // FOR...IN — parcurge CHEILE unui obiect const persoana = { nume: "Robert", varsta: 25 }; for (const cheie in persoana) { console.log(cheie, "=", persoana[cheie]); } // "nume = Robert", "varsta = 25"
🎓 Tabel comparativ bucle
for = când știi CÂTE iterații vrei (de la 0 la 10)
while = când nu știi câte iterații, dar știi CONDIȚIA
for...of = cel mai simplu mod de a parcurge un ARRAY
for...in = parcurge CHEILE unui obiect

💡 break = oprește bucla complet | continue = sare la următoarea iterație

⚙️ Lecția 5 — Funcții function, return, arrow =>

⚙️ Funcții clasice vs Arrow Functions

O funcție e un bloc de cod reutilizabil — ca o rețetă pe care o poți chema oricând.

javascript // FUNCȚIE CLASICĂ function aduna(a, b) { return a + b; } aduna(3, 5); // → 8 // ARROW FUNCTION (forma scurtă) const scade = (a, b) => a - b; scade(10, 3); // → 7 // Cu un parametru — fără paranteze const dublu = n => n * 2; dublu(21); // → 42 // Fără return (doar face ceva) function saluta(nume) { console.log(`Salut, ${nume}!`); // Nu are return → returnează undefined }
🎓 Când folosești ce?
function = pentru funcții mari, cu nume clar.
=> (arrow) = pentru funcții mici, callback-uri, și ca parametri (.map, .filter, etc.).
return e OPȚIONAL — funcția poate doar "face" ceva fără să returneze.

📋 Lecția 6 — Array-uri .map(), .filter(), .reduce(), .forEach()

📋 Metodele magice ale array-urilor

Array = o listă de elemente. Metodele de array sunt super-puteri ale JavaScript-ului!

javascript const numere = [1, 2, 3, 4, 5]; // .map() — TRANSFORMĂ fiecare element numere.map(n => n * 10); // → [10, 20, 30, 40, 50] // .filter() — FILTREAZĂ (păstrează doar ce trece testul) numere.filter(n => n > 3); // → [4, 5] // .reduce() — COMBINĂ totul într-o singură valoare numere.reduce((total, n) => total + n, 0); // → 15 // .forEach() — EXECUTĂ ceva pentru fiecare (nu returnează) numere.forEach(n => console.log(n)); // .push() / .pop() — adaugă la sfârșit / scoate ultimul numere.push(6); // [1,2,3,4,5,6] numere.pop(); // scoate 6, rămâne [1,2,3,4,5] // CHAINING — lanțuiești metode! numere .filter(n => n % 2 === 0) // [2, 4] .map(n => n * 100); // [200, 400]
💡 Tabel comparativ
.map() → Array NOU, transformând fiecare element
.filter() → Array NOU, doar elementele care trec testul
.reduce() → O SINGURĂ valoare (sumă, maxim, obiect, etc.)
.forEach() → Nimic nou, doar execută ceva pt fiecare

📝 Lecția 7 — Metode String .split(), .includes(), .slice(), .replace()

📝 String-urile au super-puteri!

Un string nu e doar text — are zeci de metode utile. Iată cele mai importante:

javascript const text = "Salut, JavaScript!"; // LUNGIME text.length; // 18 // MAJUSCULE / MINUSCULE text.toUpperCase(); // "SALUT, JAVASCRIPT!" text.toLowerCase(); // "salut, javascript!" // CĂUTARE text.includes("Java"); // true (conține "Java"?) text.startsWith("Sal"); // true text.indexOf("Java"); // 7 (poziția unde începe) // TĂIERE text.slice(0, 5); // "Salut" (de la 0 la 5) text.slice(7); // "JavaScript!" (de la 7 la capăt) // ÎNLOCUIRE text.replace("JavaScript", "Lume"); // "Salut, Lume!" // SPLIT — transformă string în ARRAY "a,b,c".split(","); // ["a", "b", "c"] "Salut lume".split(" "); // ["Salut", "lume"] // TRIM — scoate spațiile de la capete " salut ".trim(); // "salut" // REPEAT "ha".repeat(3); // "hahaha"
💡 String-urile sunt IMUTABILE!
Metodele de string NU modifică string-ul original — returnează unul NOU!
text.toUpperCase() NU schimbă text — trebuie să salvezi: const nou = text.toUpperCase();

.split() + .reverse() + .join() = inversarea unui text! Încearcă în Playground: "Robert".split("").reverse().join("")

🧱 Lecția 8 — Obiecte proprietăți, metode, this

🧱 Obiectele — structuri de date complexe

Un obiect grupează date și funcții (metode) sub un singur "capac". Ca o fișă de angajat.

javascript const persoana = { // Proprietăți (date) nume: "Robert", varsta: 25, hobbyuri: ["coding", "gaming"], // Metodă (funcție în obiect) prezentare() { // "this" = obiectul curent (persoana) return `${this.nume}, ${this.varsta} ani`; } }; persoana.nume; // "Robert" persoana.hobbyuri[0]; // "coding" persoana.prezentare(); // "Robert, 25 ani" // Destructuring — extragi proprietăți în variabile const { nume, varsta } = persoana; // Acum: nume = "Robert", varsta = 25
🎓 Ce e "this"?
this = "eu, obiectul curent". Când scrii this.nume într-o metodă, te referi la proprietatea nume a obiectului în care e metoda. E ca și cum ai zice "numele meu".

⚖️ Lecția 9 — Comparații & Logică ==, ===, &&, ||, !

⚖️ Operatori de comparare și logici

Fundația oricărui if! Cum compari valori și combini condiții.

javascript // == vs === (FOARTE IMPORTANT!) 5 == "5" // true — == convertește tipurile 5 === "5" // false — === compară ȘI tipul // 🏆 FOLOSEȘTE MEREU === ! // Operatori de comparare 10 > 5 // true (mai mare) 3 < 7 // true (mai mic) 5 >= 5 // true (mai mare sau egal) 4 <= 3 // false (mai mic sau egal) 5 !== "5" // true (strict diferit) // OPERATORI LOGICI true && true // true — AND (ambele trebuie true) true && false // false true || false // true — OR (cel puțin unul true) false || false // false !true // false — NOT (neagă) !false // true // COMBINAT — exemplu real const varsta = 20; const arePermis = true; if (varsta >= 18 && arePermis) { console.log("Poți conduce!"); }
🎓 Memorează așa
&& (AND) = "ȘI" — ambele trebuie să fie adevărate
|| (OR) = "SAU" — cel puțin una trebuie să fie adevărată
! (NOT) = "NU" — inversează: true → false, false → true

Trucuri: const x = valoare || "default" — dacă valoare e falsy, folosește "default"
const y = obj?.prop — optional chaining (nu crape dacă obj e null)

🛡️ Lecția 10 — Error Handling try, catch, finally, throw

🛡️ try/catch — Prinde erorile fără crash!

Fără try/catch, o eroare OPREȘTE tot codul. Cu try/catch, o prinzi și poți face ceva cu ea.

javascript // FĂRĂ try/catch → pagina crape // variabilaInexistenta; // ❌ ReferenceError! // CU try/catch → eroarea e prinsă try { const data = JSON.parse("nu sunt JSON"); } catch (err) { console.log("Eroare:", err.message); // → "Unexpected token n in JSON" } finally { console.log("Se execută MEREU"); } // THROW — arunci PROPRIA ta eroare function divide(a, b) { if (b === 0) { throw new Error("Nu poți împărți la zero!"); } return a / b; } try { divide(10, 0); } catch (err) { console.log(err.message); // "Nu poți împărți la zero!" }
🎓 Când folosești try/catch?
1. Când faci JSON.parse() (datele pot fi invalide)
2. Când faci fetch() către un API (rețeaua poate pica)
3. Când execuți cod de la utilizator (ca eval() în playground!)
4. Când citești fișiere sau accesezi baze de date

finally = se execută MEREU, indiferent dacă e eroare sau nu (bun pentru cleanup)

🧰 Lecția 11 — Built-in Objects Math, Date, JSON, localStorage

🧰 Obiectele built-in — deja incluse în JS!

JavaScript vine cu obiecte gata făcute, pline de funcții utile. Nu trebuie să le importi!

javascript // === MATH — operații matematice === Math.random(); // număr aleator 0-1 (ex: 0.7342) Math.floor(4.7); // 4 (rotunjește în JOS) Math.ceil(4.2); // 5 (rotunjește în SUS) Math.round(4.5); // 5 (rotunjire normală) Math.max(3, 7, 1); // 7 (maximul) Math.min(3, 7, 1); // 1 (minimul) Math.abs(-5); // 5 (valoare absolută) // Random 1-100: Math.floor(Math.random() * 100) + 1 // === DATE — data și ora === const acum = new Date(); acum.getFullYear(); // 2026 acum.getMonth(); // 0-11 (ianuarie=0!) acum.getDate(); // ziua lunii (1-31) acum.getHours(); // ora (0-23) acum.toLocaleDateString("ro-RO"); // "03.03.2026" // === JSON — convert obiecte ↔ text === const obj = { a: 1, b: 2 }; JSON.stringify(obj); // '{"a":1,"b":2}' (obiect → text) JSON.parse('{"a":1}'); // {a: 1} (text → obiect) // === localStorage — salvează date în browser! === localStorage.setItem("nume", "Robert"); // salvează localStorage.getItem("nume"); // "Robert" localStorage.removeItem("nume"); // șterge // Datele PERSISTĂ chiar dacă închizi browser-ul!
💡 localStorage — cea mai ușoară "bază de date"
Vrei să salvezi setările utilizatorului? Scorul la un joc? Codul din playground?
localStorage e cel mai simplu mod — date salvate direct în browser!

⚠️ localStorage salvează DOAR string-uri! Pentru obiecte:
localStorage.setItem("data", JSON.stringify(obiect));
const data = JSON.parse(localStorage.getItem("data"));

🔒 Lecția 12 — Scope & Closures global, local, block, closure

🔒 Scope — unde "trăiesc" variabilele

Scope = zona din cod în care o variabilă e vizibilă. Dacă o declari într-o funcție, nu o poți accesa din afară!

javascript // GLOBAL SCOPE — vizibilă ORIUNDE const global = "Sunt vizibilă peste tot!"; function exemplu() { // LOCAL SCOPE — vizibilă DOAR în funcție const local = "Exist doar aici"; console.log(global); // ✅ Merge! console.log(local); // ✅ Merge! } // console.log(local); // ❌ ERROR! local nu există aici // BLOCK SCOPE (let/const) — vizibilă doar în { } if (true) { let block = "Doar în if"; var vechi = "Scapă din if!"; // ⚠️ var NU respectă block scope! } // console.log(block); // ❌ ERROR! // console.log(vechi); // ✅ Merge (de aceea var e rău!)

🎁 Closures — funcții care "țin minte"

Un closure e o funcție care "își amintește" variabilele din mediul unde a fost creată, chiar și după ce acel mediu a terminat de executat.

javascript // CLOSURE — funcția internă "ține minte" count function creazaContor() { let count = 0; // variabilă "privată" return function() { count++; return count; }; } const contor = creazaContor(); contor(); // 1 contor(); // 2 contor(); // 3 // Nu poți accesa "count" direct — e PRIVAT! // Doar funcția returnată poate modifica count // EXEMPLU PRACTIC — funcție cu "memorie" function salutGenerator(salut) { return function(nume) { return `${salut}, ${nume}!`; }; } const salutRO = salutGenerator("Bună"); const salutEN = salutGenerator("Hello"); salutRO("Robert"); // "Bună, Robert!" salutEN("Robert"); // "Hello, Robert!"
🎓 De ce contează closures?
1. Variabile private — ascunzi date de restul codului
2. Funcții fabrică — creezi funcții personalizate
3. Event handlers — butoanele din playground folosesc closures!
4. React Hooks — useState, useEffect sunt bazate pe closures

💡 Regulă: orice funcție definită înăuntrul altei funcții are acces la variabilele funcției exterioare

📦 Lecția 13 — Destructuring & Spread extrage, combină, copiază

📦 Destructuring — extragi valori rapid

Destructuring = scurtătură pentru a extrage valori din obiecte și array-uri în variabile separate.

javascript // === OBJECT DESTRUCTURING === const persoana = { nume: "Robert", varsta: 25, oras: "București" }; // FĂRĂ destructuring (lung) const n = persoana.nume; const v = persoana.varsta; // CU destructuring (scurt!) const { nume, varsta, oras } = persoana; // Acum: nume = "Robert", varsta = 25, oras = "București" // Cu REDENUMIRE const { nume: numeleMeu, varsta: varstaMea } = persoana; // Cu VALOARE DEFAULT const { job = "nespecificat" } = persoana; // job = "nespecificat" (nu există în obiect) // === ARRAY DESTRUCTURING === const culori = ["roșu", "verde", "albastru"]; const [prima, aDoua, aTreia] = culori; // prima = "roșu", aDoua = "verde", aTreia = "albastru" // Sari peste elemente const [, , ultima] = culori; // ultima = "albastru" // ÎN FUNCȚII — super util! function afiseazaUser({ nume, varsta }) { console.log(`${nume}, ${varsta} ani`); } afiseazaUser(persoana); // "Robert, 25 ani"

✨ Spread & Rest — cele 3 puncte magice (...)

... are 2 funcții: SPREAD (desfaci) și REST (aduni). Contextul decide care e.

javascript // === SPREAD — "desfaci" un array/obiect === const numere = [1, 2, 3]; const mai_multe = [...numere, 4, 5]; // [1, 2, 3, 4, 5] // COPIERE array (fără referință!) const copie = [...numere]; // copie independentă // COMBINARE array-uri const a = [1, 2], b = [3, 4]; const combinat = [...a, ...b]; // [1, 2, 3, 4] // SPREAD pe obiecte const user = { nume: "Robert", varsta: 25 }; const userUpdated = { ...user, varsta: 26, oras: "București" }; // { nume: "Robert", varsta: 26, oras: "București" } // === REST — "aduni" restul argumentelor === function suma(...numere) { return numere.reduce((t, n) => t + n, 0); } suma(1, 2, 3, 4); // 10 (primește oricâte argumente!) // REST în destructuring const [primul, ...restul] = [1, 2, 3, 4]; // primul = 1, restul = [2, 3, 4]
🎓 Unde le vezi peste tot?
React: const [state, setState] = useState(0); → Array destructuring!
Props: function Component({ title, children }) → Object destructuring!
State updates: setState({...state, name: "nou"}) → Spread!

💡 E imposibil să scrii React/Vue/Angular fără destructuring și spread. Învață-le BINE!

🏛️ Lecția 14 — Classes (OOP) class, constructor, extends, super

🏛️ Classes — "planuri" pentru obiecte

O clasă e un plan/șablon din care creezi obiecte. Ca o matriță pentru prăjituri — faci câte vrei din ea!

javascript // DEFINIREA unei clase class Animal { // constructor = se apelează automat la "new Animal()" constructor(nume, sunet) { this.nume = nume; // proprietate this.sunet = sunet; } // Metodă (funcție a clasei) vorbeste() { return `${this.nume} face: ${this.sunet}!`; } } // CREAREA obiectelor (instanțe) const caine = new Animal("Rex", "Ham"); const pisica = new Animal("Mimi", "Miau"); caine.vorbeste(); // "Rex face: Ham!" pisica.vorbeste(); // "Mimi face: Miau!"

🧬 Moștenire — extends & super

extends = o clasă "moștenește" tot de la alta. super = apelează constructorul părintelui.

javascript // Clasa PĂRINTE class Vehicul { constructor(marca, vitezaMax) { this.marca = marca; this.vitezaMax = vitezaMax; this.vitezaCurenta = 0; } accelereaza(km) { this.vitezaCurenta = Math.min( this.vitezaCurenta + km, this.vitezaMax ); return `${this.marca}: ${this.vitezaCurenta} km/h`; } franeaza() { this.vitezaCurenta = 0; return `${this.marca}: STOP!`; } } // Clasa COPIL (moștenește de la Vehicul) class Masina extends Vehicul { constructor(marca, vitezaMax, nrUsi) { super(marca, vitezaMax); // apelează constructor-ul Vehicul this.nrUsi = nrUsi; // proprietate NOUĂ } // Metodă NOUĂ (doar pentru Masina) descriere() { return `${this.marca} cu ${this.nrUsi} uși, max ${this.vitezaMax} km/h`; } } const bmw = new Masina("BMW", 250, 4); bmw.accelereaza(100); // "BMW: 100 km/h" (moștenită!) bmw.descriere(); // "BMW cu 4 uși, max 250 km/h"
🎓 Classes vs Objects?
Obiect literal { } = bun pentru un singur obiect unic
Class = bun când ai nevoie de MULTE obiecte similare

🏭 Gândește-te la o clasă ca la o FABRICĂ:
class Masina = fabrica | new Masina("BMW") = o mașină produsă

💡 În React, componentele de clasă folosesc exact sintaxa asta!
class App extends React.Component { ... }

⏳ Lecția 15 — Promises & Async/Await cod asincron, .then(), await

⏳ Asincron — codul care NU blochează

JS e single-threaded — execută o singură linie la un moment dat. Dar unele operații durează (rețea, timere). Async = "începe asta, dar continuă cu restul, și anunță-mă când e gata!"

javascript // === setTimeout — cel mai simplu exemplu de async === console.log("1. Start"); setTimeout(() => { console.log("2. După 2 secunde"); }, 2000); console.log("3. Imediat"); // Ordinea OUTPUT: 1, 3, 2 (NU 1, 2, 3!) // JS NU așteaptă — continuă și revine când timer-ul e gata // === setInterval — repetă la fiecare X ms === let sec = 0; const timer = setInterval(() => { sec++; console.log(`${sec} secunde`); if (sec >= 5) clearInterval(timer); // STOP! }, 1000);

🤝 Promises — "promisiunea" unui rezultat

Un Promise e un obiect care spune: "Voi avea un rezultat în viitor — fie SUCCESS, fie EROARE."

javascript // CREARE Promise const promisiune = new Promise((resolve, reject) => { const succes = true; if (succes) { resolve("Datele au venit! ✅"); } else { reject("Eroare de rețea ❌"); } }); // CONSUMARE cu .then() / .catch() promisiune .then(rezultat => console.log(rezultat)) .catch(eroare => console.log(eroare)) .finally(() => console.log("Gata!")); // EXEMPLU REAL — simulăm un apel la server function getUser(id) { return new Promise((resolve, reject) => { setTimeout(() => { if (id > 0) resolve({ id, nume: "Robert" }); else reject("ID invalid!"); }, 1000); }); }

✨ Async/Await — Promises, dar MAI ELEGANT!

async/await = zahăr sintactic peste Promises. Codul ARATĂ sincron, dar e tot asincron!

javascript // CU .then() (greu de citit pe multiple apeluri) getUser(1) .then(user => console.log(user)) .catch(err => console.log(err)); // CU async/await (MULT mai curat!) async function incarcaUser() { try { const user = await getUser(1); console.log(user); // { id: 1, nume: "Robert" } } catch (err) { console.log("Eroare:", err); } } incarcaUser(); // "await" = "AȘTEAPTĂ rezultatul Promise-ului" // Poți folosi await DOAR în funcții marcate cu "async" // try/catch funcționează normal cu async/await! // MULTIPLE apeluri PARALELE async function incarcaTot() { const [user1, user2] = await Promise.all([ getUser(1), getUser(2) ]); console.log(user1, user2); }
🎓 .then() vs async/await?
Ambele fac ACELAȘI lucru — async/await e doar mai ușor de citit.
.then() = bun pentru operații simple, într-o singură linie
async/await = bun când ai MULTIPLE operații secvențiale

⚠️ Greșeala #1 a juniorilor: uită să pună await!
Fără await, primești Promise-ul, NU rezultatul!

🌍 Lecția 16 — Fetch API apeluri HTTP, GET, POST, JSON

🌍 Fetch — ia date de pe internet!

fetch() e funcția nativă JS pentru a face apeluri HTTP. Returnează un Promise!

javascript // === GET — ia date de la un API === async function getUsers() { try { // Pas 1: fetch trimite cererea HTTP const response = await fetch( "https://jsonplaceholder.typicode.com/users" ); // Pas 2: verifică dacă răspunsul e OK (status 200) if (!response.ok) { throw new Error(`HTTP ${response.status}`); } // Pas 3: convertește răspunsul din JSON în obiect JS const users = await response.json(); // Pas 4: folosește datele! users.forEach(u => { console.log(`${u.name} — ${u.email}`); }); } catch (err) { console.log("Eroare:", err.message); } } // === POST — trimite date la un API === async function createUser(nume, email) { const response = await fetch( "https://jsonplaceholder.typicode.com/users", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ name: nume, email: email }) } ); const data = await response.json(); console.log("Creat:", data); }
🎓 Pasul clasic fetch
1. fetch(url) → trimite cererea, returnează Promise
2. response.json() → convertește răspunsul (tot Promise!)
3. Folosește datele

⚠️ fetch() NU aruncă eroare la 404 sau 500! Doar la erori de rețea.
De aceea verifici response.ok manual!

💡 Încearcă în Playground: fetch("https://jsonplaceholder.typicode.com/todos/1").then(r => r.json()).then(d => console.log(d))

🎯 Lecția 18 — Events & Listeners addEventListener, click, keydown, input

🎯 Cele 3 metode de a asculta evenimentele

Un event = ceva ce se întâmplă pe pagină: click, tastă apăsată, mouse mișcat, form trimis.

javascript // ❌ Metoda 1: INLINE (NU recomand) // <button onclick="alert('click')"> // Probleme: amesteci HTML cu JS, greu de depanat // ⚠️ Metoda 2: PROPRIETATE (limitată) // btn.onclick = function() { ... }; // Problemă: poți pune DOAR UN handler per event // ✅ Metoda 3: addEventListener (RECOMANDATĂ!) btn.addEventListener("click", () => { console.log("Click!"); }); // ✅ Poți pune MULTIPLE handlere pe același event // ✅ Poți scoate handler-ul cu removeEventListener // OBIECTUL EVENT (e) input.addEventListener("input", (e) => { e.target // elementul pe care s-a întâmplat e.target.value // valoarea curentă a input-ului e.key // tasta apăsată (la keydown) e.preventDefault() // oprește comportamentul default });
🎓 De ce addEventListener e cea mai bună?
Poți pune mai mulți listeneri pe același element. Poți scoate un listener specific. Și cel mai important: separi complet JavaScript-ul de HTML. Deschide script.js și caută #ADDEVENTLISTENER!

🛑 preventDefault() — Oprește browser-ul

Uneori vrei să oprești ce face browser-ul automat (ex: submit reîncarcă pagina, link navighează).

javascript // Oprește un formular să reîncarce pagina form.addEventListener("submit", (e) => { e.preventDefault(); // Acum procesezi datele cu JS, fără reload }); // Oprește un link să navigheze link.addEventListener("click", (e) => { e.preventDefault(); console.log("Link blocat!"); });
🎓 Când folosești preventDefault?
Cel mai des la formulare (să nu reîncarce pagina) și la link-uri (să faci altceva în loc). În script.js, caută #PREVENTDEFAULT — e folosit la navigarea cu ⬆️⬇️ în consolă!
