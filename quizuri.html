Răspunsuri corecte
📦 Variabile
1. Ce se întâmplă dacă faci const x = 5; x = 10;?
A x devine 10
B TypeError — const nu poate fi reasignat
C x devine undefined
D Nu se întâmplă nimic
✅ Corect! const nu permite reasignarea. Dacă vrei să schimbi valoarea, folosește let.
❌ Greșit! const = CONSTANT. Nu poți face x = 10 dacă l-ai declarat cu const. Ai primi TypeError!
🏷️ Tipuri
2. Ce returnează typeof null?
A "null"
B "undefined"
C "object"
D "boolean"
✅ Corect! typeof null returnează "object" — e un bug din 1995 care NU a fost reparat niciodată!
❌ Surpriză! typeof null returnează "object", NU "null". E un bug celebru din 1995!
⚖️ Comparații
3. Ce returnează 5 === "5"?
A true
B false
C undefined
D Error
✅ Corect! === (strict equality) compară ȘI tipul. 5 este number, "5" este string → false.
❌ === verifică ȘI tipul! 5 (number) !== "5" (string). Doar == ar da true (convertește tipurile).
📋 Array-uri
4. Ce returnează [1,2,3].map(n => n * 2)?
A [2, 4, 6]
B 12
C [1, 2, 3, 1, 2, 3]
D undefined
✅ Corect! .map() aplică funcția pe FIECARE element și returnează un array NOU: [1*2, 2*2, 3*2] = [2, 4, 6].
❌ .map() transformă fiecare element: 1→2, 2→4, 3→6. Rezultat: [2, 4, 6].
⚙️ Funcții
5. Care e diferența dintre function f() {} și const f = () => {}?
A Nu e nicio diferență
B Arrow function nu are propriul "this"
C Arrow function e mai rapidă
D Arrow function nu poate avea parametri
✅ Corect! Arrow functions NU au propriul this — moștenesc this din contextul exterior. De aceea nu le folosești ca metode în obiecte.
❌ Diferența principală: arrow functions NU au propriul this. Moștenesc this din contextul unde au fost definite.
🔀 Condiții
6. Ce afișează console.log(true && "salut")?
A true
B false
C "salut"
D undefined
✅ Corect! && returnează ULTIMA valoare dacă ambele sunt truthy. true && "salut" → "salut".
❌ && NU returnează true/false neapărat! Returnează ultima valoare truthy dacă ambele sunt truthy: true && "salut" → "salut".
🔄 Bucle
7. De câte ori rulează for (let i = 0; i < 3; i++) {}?
A 2 ori (0, 1)
B 3 ori (0, 1, 2)
C 4 ori (0, 1, 2, 3)
D Infinit
✅ Corect! i=0 ✓, i=1 ✓, i=2 ✓, i=3 ✗ (3 < 3 e false). Total: 3 iterații.
❌ Condiția e i < 3: i=0 ✓, i=1 ✓, i=2 ✓, i=3 ✗ (stop). Total: 3 ori.
📝 String-uri
8. Ce returnează "JavaScript".slice(0, 4)?
A "Java"
B "JavaS"
C "Jav"
D "Script"
✅ Corect! .slice(0, 4) = de la index 0 la index 4 (exclusiv). J(0) a(1) v(2) a(3) = "Java".
❌ .slice(start, end) — end e EXCLUSIV! Index 0,1,2,3 = "Java" (4 caractere, se oprește ÎNAINTE de index 4).
🔒 Scope
9. Ce se întâmplă cu variabila declarată cu let într-un if { } block?
A E vizibilă peste tot în fișier
B E vizibilă DOAR în acel { } block
C E vizibilă în toată funcția
D E vizibilă doar pe linia curentă
✅ Corect! let și const au BLOCK SCOPE — vizibile doar între { }. Asta le face mai sigure ca var!
❌ let are BLOCK SCOPE — vizibilă doar între { }. var ar fi vizibilă în toată funcția (function scope).
📦 Destructuring
10. Ce face const { name, age } = person;?
A Creează un obiect nou
B Extrage name și age din obiectul person
C Șterge name și age din person
D Dă eroare de sintaxă
✅ Corect! Destructuring extrage proprietăți din obiecte în variabile separate. E ca și cum ai scrie: const name = person.name; const age = person.age;
❌ E DESTRUCTURING: extrage name și age din person în variabile separate. Nu modifică și nu șterge nimic!
⏳ Async
11. Ce ordine afișează: console.log("A"); setTimeout(() => console.log("B"), 0); console.log("C");
A A, B, C
B A, C, B
C B, A, C
D C, B, A
✅ Corect! setTimeout pune callback-ul la COADĂ, chiar și cu delay 0. JS execută mai întâi tot codul sincron (A, C), apoi cele din coadă (B).
❌ Chiar și cu delay 0ms, setTimeout e ASINCRON! JS execută: A (sincron), C (sincron), apoi B (din coadă). Ordinea: A, C, B.
🏛️ Classes
12. Ce face super() într-o clasă copil?
A Creează un obiect complet diferit
B Apelează constructor-ul clasei PĂRINTE
C Șterge clasa copil
D Dă eroare mereu
✅ Corect! super() apelează constructorul clasei părinte, transmițându-i parametrii. E OBLIGATORIU dacă folosești extends!
❌ super() = apelează constructor-ul clasei PĂRINTE. E obligatoriu în clase copil care folosesc extends!
🛡️ Error Handling
13. Ce se întâmplă în blocul finally?
A Se execută DOAR dacă e eroare
B Se execută DOAR dacă NU e eroare
C Se execută MEREU, indiferent de rezultat
D Nu se execută niciodată
✅ Corect! finally se execută MEREU — și la succes, și la eroare. Perfect pentru cleanup (închidere fișiere, conexiuni, etc.).
❌ finally se execută MEREU! Indiferent dacă try reușește sau catch prinde o eroare. E perfect pentru cleanup.
🌍 Fetch
14. Ce returnează fetch(url)?
A Un obiect JSON direct
B Un string cu datele
C Un Promise
D Un array
✅ Corect! fetch() returnează un Promise! De aceea trebuie .then() sau await. Și response.json() returnează tot un Promise!
❌ fetch() returnează un PROMISE! Nu primești datele direct — trebuie await fetch() sau .then() pentru a obține răspunsul.
🌐 DOM
15. Care e diferența dintre textContent și innerHTML?
A Nu e nicio diferență
B textContent = text simplu, innerHTML = permite HTML
C innerHTML e mai rapid
D textContent nu funcționează în Chrome
✅ Corect! textContent = text simplu (SAFE). innerHTML = HTML (tag-uri funcționează, dar atenție la XSS!).
❌ textContent = text simplu (sigur). innerHTML = interpretează HTML (tag-uri ca <b> funcționează). Folosește textContent pentru securitate!
