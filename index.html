1. Variabile // let, const, var

🎓 Sfat de Prof
Deschide DevTools în browser (apasă F12 → tab-ul „Console"). Scrie let x = 10 și apoi x — vezi cum îți returnează valoarea? Asta e consola reală — cel mai bun prieten al unui developer. Tot ce vezi mai jos poți testa și acolo!

📦 let & const

let — variabilă care poate fi reasignată.
const — valoare constantă, nu poate fi schimbată.

js let varsta = 25; varsta = 26; // ✅ OK const nume = "Robert"; // nume = "Ion"; ❌ Error!

🏷️ Tipuri de date

JavaScript are tipuri primitive: string, number, boolean, null, undefined.

js let text = "Salut"; // string let nr = 42; // number let activ = true; // boolean let gol = null; // null let x; // undefined

2. Funcții // function, arrow =>

⚙️ Funcție clasică

Funcțiile sunt blocuri de cod reutilizabile. Se declară cu function.

js function salut(nume) { return `Bună, ${nume}!`; } console.log(salut("Robert"));

➡️ Arrow Function

Sintaxă modernă și concisă, introdusă în ES6. Perfectă pentru funcții scurte.

js const dublu = (n) => n * 2; console.log(dublu(7)); // 14 const sumă = (a, b) => a + b; console.log(sumă(3, 5)); // 8

3. Arrays // liste de elemente

📋 Operații cu Arrays

Array-urile sunt colecții ordonate. Poți adăuga, șterge, filtra și transforma elementele.

js const fructe = ["măr", "banană", "portocală"]; fructe.push("kiwi"); // adaugă la sfârșit fructe.pop(); // șterge ultimul // Iterare cu forEach fructe.forEach(f => console.log(f)); // Map — transformă fiecare element const mari = fructe.map(f => f.toUpperCase()); // Filter — filtrează const scurte = fructe.filter(f => f.length <= 5);

5. TypeScript Basics // tipuri statice

🔷 De ce TypeScript?

TypeScript adaugă tipuri statice peste JavaScript. Asta înseamnă că prinzi erorile înainte să rulezi codul!

ts // TypeScript — tipuri explicite let nume: string = "Robert"; let varsta: number = 25; let activ: boolean = true; // Funcție cu tipuri function salut(nume: string): string { return `Salut, ${nume}!`; }

📐 Interfaces

Interfețele definesc forma unui obiect — ce proprietăți are și ce tipuri au.

ts interface Student { nume: string; varsta: number; limbaje: string[]; activ?: boolean; // opțional } const eu: Student = { nume: "Robert", varsta: 25, limbaje: ["JS", "TS"] };

6. DOM — Document Object Model // controlează pagina

🎓 Sfat de Prof
DOM-ul e scheletul paginii tale HTML, dar în formă de obiect JavaScript. Fiecare tag (<div>, <p>, <button>) devine un nod pe care îl poți modifica din JS. Deschide F12 → „Elements" și dă hover pe elemente — vezi cum se evidențiază pe pagină? Asta e DOM-ul!

🔎 Selectare elemente

Ca să modifici ceva pe pagină, întâi trebuie să-l găsești. Ai mai multe metode:

js // După ID — returnează UN singur element const titlu = document.getElementById("titlu"); // După selector CSS — primul găsit const btn = document.querySelector(".btn"); // Toate elementele care se potrivesc const paragrafe = document.querySelectorAll("p");
👀 Uită-te în cod
Observă: getElementById returnează un singur element, dar querySelectorAll returnează o listă (NodeList). E ca diferența dintre „dă-mi cartea aia" vs „dă-mi toate cărțile albastre".

✏️ Modificare elemente

Odată selectat un element, poți schimba textul, stilul sau atributele.

js // Schimbă textul titlu.textContent = "Text nou!"; // Schimbă HTML-ul interior titlu.innerHTML = "<em>Italic!</em>"; // Schimbă stilul CSS titlu.style.color = "#00d2ff"; titlu.style.fontSize = "2rem"; // Adaugă/scoate clase CSS titlu.classList.add("activ"); titlu.classList.toggle("vizibil");
⚠️ Atenție
textContent e safe — pune text simplu. innerHTML interpretează HTML — poate fi periculos dacă vine de la utilizator (XSS). Începe cu textContent mereu!

7. Events — Handlers & Listeners // reacționează la utilizator

🎓 Sfat de Prof
Un event e orice acțiune: click, hover, tastare, scroll, resize. JavaScript poate asculta aceste evenimente și reacționa. Gândește-te la asta ca la un recepționer la hotel: stă și ascultă (listener), iar când cineva vine (event), reacționează (handler). Sunt 3 moduri de a face asta — de la simplu la profesional:

1️⃣ Inline Event Handler — evită pe cât posibil

Pui codul direct în HTML, în atributul onclick. E simplu dar amestecă HTML cu JS — nu e clean!

html <!-- ❌ Nu e ideal — amesteci HTML cu logica --> <button onclick="alert('Ai dat click!')">Click</button>
👀 Uită-te în cod
Butoanele de mai sus (din demo-ul DOM) folosesc exact asta — onclick="...". Funcționează, dar imaginează-ți 50 de butoane, fiecare cu cod inline... Dezastru! De-aia avem metode mai bune. 👇

2️⃣ Event Handler Property — OK pentru cazuri simple

Setezi proprietatea .onclick direct pe element din JavaScript. Mai curat, dar poți pune doar UN handler per event.

js const btn = document.querySelector("#myBtn"); btn.onclick = function() { console.log("Click!"); }; // ⚠️ Atenție: asta ÎNLOCUIEȘTE handler-ul de sus! btn.onclick = function() { console.log("Alt click!"); }; // Doar "Alt click!" va rula

3️⃣ addEventListener — ✅ METODA CORECTĂ

Cea mai flexibilă. Poți adăuga multiple handler-uri pe același event, le poți și scoate.

js const btn = document.querySelector("#myBtn"); // Adaugă un listener btn.addEventListener("click", function() { console.log("Click 1!"); }); // Poți adăuga MAI MULȚI pe același event! btn.addEventListener("click", function() { console.log("Click 2!"); }); // Ambele rulează! 🎉 // Cu arrow function btn.addEventListener("click", () => { console.log("Click cu arrow!"); });
🎓 Sfat de Prof
addEventListener e standardul din industrie. Orice framework (React, Angular, Vue) folosește exact acest principiu. Dacă înveți o singură metodă, asta e. Tot ce urmează o folosește!

⌨️ input — text în timp real

Se declanșează la fiecare caracter tastat într-un input.

Demo Interactiv
Textul tău: nimic încă
js const input = document.getElementById("liveInput"); const output = document.getElementById("liveInputOutput"); input.addEventListener("input", (e) => { output.textContent = `Ai scris: "${e.target.value}" (${e.target.value.length} caractere)`; });
🎓 Ce e e (event object)?
Funcția ta de handler primește automat un parametru — obiectul event (îl numim e sau event). El conține informații despre ce s-a întâmplat: e.target = elementul pe care s-a întâmplat, e.target.value = valoarea curentă a input-ului. Întotdeauna ai acces la acest obiect!

🎨 mouseover / mouseout — hover effects

Se declanșează când mouse-ul intră sau iese de pe un element.

js const box = document.getElementById("hoverBox"); box.addEventListener("mouseover", () => { box.style.background = "#00d2ff"; box.style.transform = "scale(1.2) rotate(10deg)"; }); box.addEventListener("mouseout", () => { box.style.background = "#e94560"; box.style.transform = "scale(1)"; });
🎓 CSS vs JS pentru hover?
Dacă vrei doar să schimbi culoarea la hover, folosește CSS (:hover) — e mai rapid. Dar dacă ai nevoie de logică (de ex: incrementezi un counter, faci un fetch), atunci folosești JS cu addEventListener. Regula: style → CSS, logică → JS.

🛑 preventDefault — oprește comportamentul default

Unele elemente au comportamente built-in (linkurile navighează, formularele se trimit). preventDefault() le oprește!

js // Oprește un link din a naviga const link = document.querySelector("a"); link.addEventListener("click", (e) => { e.preventDefault(); console.log("Link-ul NU a navigat!"); }); // Oprește un form din a se trimite const form = document.querySelector("form"); form.addEventListener("submit", (e) => { e.preventDefault(); // Validezi datele manual aici });
🎓 Când folosești asta?
Foarte des! De exemplu: ai un formular și vrei să validezi datele cu JS înainte de a-l trimite. Fără preventDefault(), formularul se trimite instant și pagina se reîncarcă. Cu el, tu preiei controlul!
