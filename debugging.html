🐛 Debugging Tutorial

Cele 10 cele mai comune erori JavaScript — cum arată, de ce apar, și cum le repari.

10 Erori Comune
🎓 Sfat de Prof
Erorile NU sunt dușmanul tău — sunt profesorul tău! Fiecare eroare îți spune exact ce e greșit. Citește mesajul de eroare cu atenție, conține TOATĂ informația de care ai nevoie.
Bug #1

❌ ReferenceError: x is not defined

❌ Uncaught ReferenceError: myVariable is not defined
// ❌ COD GREȘIT
console.log(myVariable); // variabila nu există!
// ✅ COD CORECT
const myVariable = "Salut!";
console.log(myVariable); // → "Salut!"
🔧 Cauza: Folosești o variabilă care nu a fost declarată. Verifică: 1) Ai declarat-o cu let/const? 2) Ai scris numele corect (case-sensitive)? 3) E în scope-ul potrivit?
Bug #2

❌ TypeError: Cannot read properties of null

❌ Uncaught TypeError: Cannot read properties of null (reading 'textContent')
// ❌ COD GREȘIT
const el = document.getElementById("inexistent");
el.textContent = "test"; // el este NULL!
// ✅ COD CORECT
const el = document.getElementById("inexistent");
if (el) {
    el.textContent = "test";
} else {
    console.log("Elementul nu există!");
}
🔧 Cauza: getElementById returnează NULL dacă ID-ul nu există. Verifică MEREU cu if(el) înainte de a-l folosi! Sau verifică dacă id-ul din HTML se potrivește exact.
Bug #3

❌ TypeError: x is not a function

❌ Uncaught TypeError: myVar is not a function
// ❌ COD GREȘIT
const myVar = "text";
myVar(); // un string NU e funcție!
// ✅ COD CORECT
const myFunc = () => "text";
myFunc(); // → "text"
🔧 Cauza: Încerci să apelezi cu () ceva ce NU e funcție. Verifică: 1) Tipul variabilei (typeof x). 2) Ai scris numele corect? 3) Ai pus paranteze unde nu trebuia?
Bug #4

❌ SyntaxError: Unexpected token

❌ Uncaught SyntaxError: Unexpected token '}'
// ❌ COD GREȘIT
function salut(name {  // lipsește )
    return "Bună, " + name;
}
// ✅ COD CORECT
function salut(name) {  // ) adăugat
    return "Bună, " + name;
}
🔧 Cauza: Cod invalid sintactic. Cele mai comune: 1) Paranteze ( ) lipsă sau nepereche. 2) Acolade { } nepereche. 3) Virgule , lipsă. 4) Ghilimele " " neînchise.
Bug #5

⚠️ == vs === (bug silențios!)

⚠️ Nu e eroare, dar dă rezultate neașteptate!
// ⚠️ COD PERICULOS
5 == "5"     // true (convertește tipuri!)
0 == ""      // true (?!)
null == undefined // true (?!?!)
// ✅ COD SIGUR
5 === "5"    // false (tip diferit!)
0 === ""     // false
null === undefined // false
🔧 Regulă: Folosește MEREU === (strict equality). == face conversii automate de tip care pot cauza bug-uri subtile și greu de găsit.
Bug #6

❌ Infinite Loop — Pagina se blochează

⚠️ Pagina nu mai răspunde (browser-ul se blochează)
// ❌ COD GREȘIT — BUCLĂ INFINITĂ!
let i = 0;
while (i < 10) {
    console.log(i);
    // LIPSEȘTE i++ !!
}
// ✅ COD CORECT
let i = 0;
while (i < 10) {
    console.log(i);
    i++; // incrementează!
}
🔧 Cauza: while/for fără condiție de ieșire. Verifică: 1) while are condiție care devine false? 2) for are incrementare (i++)? 3) Ai break unde trebuie?
Bug #7

⚠️ Async/Await fără await

⚠️ Primești Promise {<pending>} în loc de date!
// ❌ COD GREȘIT
const data = fetch("https://api.example.com");
console.log(data); // Promise {<pending>} !
// ✅ COD CORECT
const response = await fetch("https://api.example.com");
const data = await response.json();
console.log(data); // Datele reale!
🔧 Cauza: fetch() returnează un Promise, nu datele direct. Trebuie: 1) await fetch() 2) await response.json() 3) Funcția trebuie să fie async.
Bug #8

⚠️ this în arrow functions

⚠️ this e undefined sau window în loc de obiectul tău!
// ❌ COD GREȘIT
const obj = {
    name: "Robert",
    greet: () => {
        console.log(this.name); // undefined!
    }
};
// ✅ COD CORECT
const obj = {
    name: "Robert",
    greet() {  // metoda normală
        console.log(this.name); // "Robert"
    }
};
🔧 Cauza: Arrow functions NU au propriul this — moștenesc pe cel din context. Pentru metode de obiect, folosește funcții normale (nu arrow).
Bug #9

⚠️ Array mutation fără să vrei

⚠️ Array-ul original e modificat când nu voiai!
// ❌ COD PERICULOS
const orig = [3, 1, 2];
const sorted = orig.sort(); // ATENȚIE!
console.log(orig); // [1, 2, 3] — MODIFICAT!
// ✅ COD CORECT
const orig = [3, 1, 2];
const sorted = [...orig].sort(); // copie!
console.log(orig);   // [3, 1, 2] — neatins!
console.log(sorted); // [1, 2, 3]
🔧 Cauza: .sort(), .reverse(), .splice() MODIFICĂ array-ul original! Folosește [...arr] (spread) pentru a face o copie înainte de modificare.
Bug #10

⚠️ 0.1 + 0.2 !== 0.3 (floating point)

⚠️ 0.1 + 0.2 = 0.30000000000000004 (?!)
// ⚠️ CIUDAT DAR ADEVĂRAT
console.log(0.1 + 0.2);         // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3); // false! 😱
// ✅ SOLUȚIE
// Rotunjește la N zecimale
console.log((0.1 + 0.2).toFixed(1)); // "0.3"
// Sau compară cu toleranță
Math.abs(0.1 + 0.2 - 0.3) < 0.0001  // true
🔧 Cauza: Numerele cu virgulă (float) NU sunt exacte în computere — e o limitare IEEE 754 care apare în TOATE limbajele, nu doar JS!
🏆 Sfat Final
Când primești o eroare, CITEȘTE MESAJUL! Conține: Tipul erorii (ReferenceError, TypeError, etc.) + Descrierea (ce e greșit) + Linia (unde e greșit). Cu aceste 3 informații, 90% din erori se rezolvă singure! 🔧
